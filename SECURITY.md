# Security Policy

如果你发现了安全问题，请不要直接公开提交 issue。

建议通过私下方式反馈，至少请附上：

- 问题描述
- 复现步骤
- 影响范围
- 是否涉及 token、账号、权限绕过或本地命令执行

在公开修复前，请不要把可直接利用的细节写进公开 issue 或 PR。

## 当前特别需要注意的点

- 不要提交真实微信登录态
- 不要提交本地 `account.json`、`.mcp.json` 或 `.claude/` 配置
- 不要在日志、截图或 issue 里暴露 `context_token`、bot token 或用户标识
- 外部消息桥接到本地执行环境时，默认优先使用 `worker` 主路径，并保持授权边界清晰
