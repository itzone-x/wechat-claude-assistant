# v0.1.0 Release Notes

发布日期：2026-03-26

## 这次发布是什么

这是项目第一次可以稳定给别人安装和试用的版本。

它不再把“微信消息桥接到当前 Claude Code 会话”当成唯一主路径，而是明确改成：

- 主路径：`worker-first`
- 高级模式：`channels`

也就是说，普通用户现在不需要一直开着一个 Claude Code 终端会话，直接在本机启动后台 worker，就可以通过微信给本地 Claude Code 派任务。

## 这次发布解决了什么问题

之前的原型更像一个 Channels 研究样例，适合技术用户验证协议，不适合稳定日用。这个版本主要解决了下面几类问题：

- 让产品主路径从“研究型桥接”变成“可安装、可后台运行、可开机自启的本地工具”
- 让微信任务和 Claude Code 子进程执行形成稳定闭环
- 让同一微信会话下的任务串行执行，不再互相打架
- 让后台运行和 `launchd` 自动启动在 macOS 上可用
- 让 worker 模式和高级 Channels 模式互相隔离，不再互相污染

## 本次重点能力

### 1. 默认使用微信派活 worker

推荐路径：

```bash
npm install
npm run build
node dist/cli.js install
node dist/cli.js start
```

你可以在微信里直接发送任务，worker 会：

1. 收到任务
2. 回传阶段状态
3. 调用本地 Claude Code 执行
4. 把结果自动回发到微信

### 2. 支持后台常驻与自动启动

支持两种常驻方式：

```bash
node dist/cli.js start --daemon
node dist/cli.js service install
```

这使得“通勤路上从微信派活，到公司再验收本地代码”成为产品的标准用法，而不是演示场景。

### 3. 会话恢复与串行控制更加稳定

本版本修复了两类关键稳定性问题：

- 旧实现重复使用 `--session-id`，会触发 `Session ID is already in use`
- 同一会话的第二条任务在真实后台环境中没有稳定被拒绝

现在的语义是：

- 新任务新会话：`--session-id`
- 已有会话恢复：`--resume`
- 同一微信会话在已有活动任务时：直接拒绝第二条任务

### 4. worker 模式不再误调用项目 MCP 工具

之前 worker 任务有可能看到项目里的微信 MCP 工具，并试图自己再调用 `wechat_reply`，导致多余的“请授权工具权限”提示。

现在 worker 子进程会强制使用空 MCP 配置，确保：

- worker 只负责产出文字结果
- 微信回传由 worker 框架自己完成
- 高级 Channels 工具不会污染主路径

## 面向使用者的变化

### 适合谁

- 想从微信直接给本地 Claude Code 派任务的人
- 不想长期维持一个交互式 Claude Code 会话的人
- 希望产品安装流程尽量接近“扫码即可用”的人

### 不适合谁

- 想把微信消息实时注入当前 Claude Code 终端会话的人
- 明确要做 `claude/channel` 研究或插件开发的人

这类需求仍然支持，但属于高级 `channels` 模式。

## 已验证内容

本版本已经做过：

- 自动化测试：`67/67`
- 真实微信登录
- 真实微信 `/echo`
- 真实微信任务执行与回传
- 真实后台 worker 运行
- 真实 `launchd` 自动启动
- 真实同会话并发拒绝
- 真实图片与语音输入联调
- 真实重复回复问题修复验证

## 已知边界

- `channels` 仍然是高级模式，不是默认路径
- `channels` 的可用性仍受 Claude Code 认证方式限制
- 当前主要面向 macOS 本地运行场景，`launchd` 也是按 macOS 实现

## 升级建议

如果你之前用的是旧原型，建议直接按下面顺序更新：

```bash
npm install
npm test
node dist/cli.js service uninstall
node dist/cli.js service install
```

如果你只想先跑前台：

```bash
node dist/cli.js start
```

如果你要停掉自动启动服务：

```bash
node dist/cli.js stop
```

如需恢复自动启动：

```bash
node dist/cli.js service install
```
