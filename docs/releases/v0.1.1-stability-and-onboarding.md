# v0.1.1 Release Notes

`v0.1.1` 是一次面向真实日常使用的补强版本，重点不在大改架构，而在把主路径的可用性、安装体验和多模态输入稳定性补扎实。

## 本版重点

- 支持文本、图片、图片链接和语音输入的 worker 主路径已经打通，并完成了真实联调。
- 消息去重、会话恢复、launchd 状态判断等影响日常使用的问题已经修复。
- 安装向导对新用户更友好，自动启动路径会主动安装服务并给出精简状态摘要。

## 主要变化

### 多模态输入

- 新增图片理解能力，支持微信原生图片和图片链接。
- 新增语音输入能力，优先结合微信自动转写，必要时附带原始音频文件。
- 支持“图片 + 文字”“语音 + 文字”的混合理解。

### 稳定性

- 修复重复投递导致的重复回复问题。
- 修复跨项目误复用旧 Claude 会话的问题。
- 修复失效 session 导致的 `No conversation found with session ID` 错误，worker 会自动重建会话后重试。
- 修复 `launchd` 场景下 `status` 对运行态的误判。

### 安装与启动体验

- 安装向导仍然默认走 `worker-first` 主路径。
- 如果用户在安装过程中选择“开启自动启动”，安装结束后会直接尝试安装并启动本机服务。
- 服务成功启动后，会立即打印精简版状态摘要，减少“装完但不知道有没有真的跑起来”的困惑。

## 推荐验证方式

安装完成后，建议至少做一次：

```bash
node dist/cli.js service status
node dist/cli.js status
```

确认：

- `已加载: 是`
- `worker 运行中: 是`

然后在微信里发送：

```text
/echo 你好
```

如果收到回复，说明主链路已经通了。
