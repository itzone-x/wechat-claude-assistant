# Code of Conduct

这个项目欢迎理性、直接、建设性的讨论。

我们希望这里保持下面几条基本原则：

- 对事不对人
- 尊重不同经验背景的参与者
- 提问题时尽量给上下文
- 提建议时尽量给理由和可执行方向
- 不发布人身攻击、骚扰、歧视或恶意内容

## 我们不欢迎

- 人身攻击
- 阴阳怪气式的恶意表达
- 明知会暴露他人隐私、账号、token 或安全细节的公开行为
- 脱离上下文的指责式评论

## 我们鼓励

- 用 issue 清楚描述问题
- 用 PR 清楚描述变更和验证方式
- 针对 `worker-first` 主路径优先提出高价值改进
- 对稳定性、易用性和安全边界提出具体建议

如果讨论已经偏离问题本身，维护者有权关闭相关讨论或限制互动方式。
